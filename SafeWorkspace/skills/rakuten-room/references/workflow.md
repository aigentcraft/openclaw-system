# 楽天ROOM ワークフロー詳細手順

## 朝の投稿ワークフロー（9:00 JST）

### Phase 1: データ収集

```bash
cd workspace/rakutenROOM_automation
```

**1-1. ランキング取得（必須）**
```bash
python3 scripts/collect_ranking.py
```
- 出力: `data/ranking_YYYY-MM-DD.json`
- 楽天API経由で上位50商品を取得
- 失敗時フォールバック: 前日のランキングJSONを使用

**1-2. トレンド収集（オプション・週2-3回）**
```bash
python3 scripts/collect_trends.py
```
- 出力: `data/trends_YYYY-MM-DD.json`
- ブラウザ操作が必要なため、失敗しやすい。失敗してもスキップ可

**1-3. 競合分析（オプション・週1-2回）**
```bash
python3 scripts/collect_competitor.py
```
- 出力: `data/knowledge_raw_YYYY-MM-DD.json`
- learning.dbに運用知見を蓄積
- 時間がかかるため毎日は不要

### Phase 2: 分析

**2-1. 戦略分析（必須）**
```bash
python3 scripts/analyze_strategy.py
```
- 入力: ランキングJSON + learning.db + 過去の実行結果
- 出力: `data/strategy_YYYY-MM-DD.json`
- 推奨カテゴリ、ターゲット層、投稿テンプレートを生成

### Phase 3: 計画生成

**3-1. 投稿計画生成（必須）**
```bash
python3 scripts/generate_post_plan.py
```
- 入力: 戦略分析 + ランキング + learning.db
- 出力: `data/post_plan_YYYY-MM-DD.json`
- 3-5商品の候補を生成（ここから1-2件を選定）

**3-2. 季節・セール情報の確認**

投稿計画を評価する際、以下を確認:
- 今日は「5と0のつく日」か？ → ポイント倍率を訴求
- 楽天スーパーSALE / お買い物マラソン期間中か？ → セール対象商品を優先
- 季節イベント（新生活、母の日、夏物等）に合う商品か？

確認方法:
- ブラウザで https://event.rakuten.co.jp/ を確認
- または楽天市場トップページのバナーを確認

### Phase 4: ユーザー確認（必須）

**投稿計画をユーザーに提示する。以下の情報を含める:**

```
📝 本日の投稿計画

【投稿タイプ】単品紹介 / まとめ○選
【商品】
1. 商品名 - ¥価格 (ショップ名)
   選定理由: ○○
   紹介文案: ○○
   ハッシュタグ: #○○ #○○
2. (まとめの場合、追加商品)

【画像スタイル】
- メインコピー: 「○○」
- サブコピー: 「○○」
- レイアウト: 単品アップ / コラージュ

【タイミング根拠】
- セール情報: ○○
- 季節性: ○○

承認しますか？修正点があれば教えてください。
```

**ユーザーが承認するまで投稿に進まないこと。**

### Phase 5: 画像加工

承認後、投稿用画像を作成する。

**5-1. 商品画像・詳細の取得**
```bash
python3 scripts/collect_product_detail.py --url "商品URL"
```
- 商品ページのスクリーンショット + 画像ダウンロード
- Vision APIで商品の特徴を分析
- Geminiでプロモーション画像を生成

**5-2. テキストオーバーレイ加工**

Gemini生成画像 or 商品画像に以下を追加:
- **大きな太字コピー**: 商品の最大の魅力（2-3行）
  - 例: 「割引えぐっ 2度見しないなんて無理すぎ」
  - 例: 「リピ確定！もう手放せない」
  - 例: 「30%OFFクーポン使える」
- **絵文字**: ✨🫶💕🙌 等を自然に配置
- **価格情報**: セール価格、割引率を目立つ位置に
- **アカウント名**: 「@Ayaka」をウォーターマークとして
- **検索誘導**: 「楽天ROOMで"○○"で検索🔍」

テキストオーバーレイの実装方法:
1. Geminiで画像にテキスト入りの画像を直接生成する（推奨）
2. またはPIL/Pillowでテキスト合成する

**5-3. まとめ/○選形式の場合**

複数商品のコラージュ画像を作成:
- 各商品画像を均等に配置（2x2、2x3等のグリッド）
- タイトルバナー: 「QOL爆上がり！厳選○選」等
- 各商品に番号と短い説明を添える
- 全体のカラートーンを統一（暖色系、ナチュラル系等）

### Phase 6: 投稿実行（承認後のみ）

**6-1. 楽天ROOMにログイン**
- ブラウザで https://myroom.rakuten.co.jp にアクセス
- ログイン状態を確認、未ログインなら認証

**6-2. 商品を投稿**
- 商品URLをペースト
- 紹介文を入力（承認された文面）
- ハッシュタグを追加
- 加工した画像をアップロード（オリジナル写真として）
- 投稿ボタンを押す

**6-3. 投稿確認**
- 投稿が正常に公開されたか確認
- learning.dbに結果を記録

**6-4. まとめ/○選の場合**
- 各商品を個別に投稿 + コレクションにまとめる
- または1つの投稿にまとめ画像 + 全商品リンク

### 投稿間の間隔

- 複数投稿する場合: 2-5分の間隔を空ける
- 投稿後30分以上経ってから次の活動（いいね等）を行う

## 夕方のエンゲージメントワークフロー（18:00 JST）

### 目的
同ジャンルのユーザーと自然な交流を行い、アカウントの露出を増やす。

### 手順

**1. 楽天ROOMを開く**
- ブラウザで https://room.rakuten.co.jp にアクセス
- ログイン状態を確認

**2. 同ジャンルユーザーを探す**
- 「インテリア」「生活雑貨」「キッチン」等のカテゴリを閲覧
- 人気の投稿やおすすめユーザーをチェック
- フォロワー数が近い（同規模）のアカウントを優先

**3. いいね活動（5-10件）**
- 良いと思った投稿にいいねする
- 機械的に連打せず、投稿を1つずつ見て判断
- 各いいねの間に3-8秒の間隔
- 同一ユーザーへの連続いいねは3件まで

**4. コメント活動（2-3件）**
- 特に共感できる投稿に短いコメントを残す
- コメント例:
  - 「これ気になってました！素敵なレビューですね✨」
  - 「わぁ、こんな使い方があるんですね！参考になります🙌」
  - 「色味がとっても素敵...！お部屋に合いそう💕」
- 不自然な定型文は避ける。投稿内容に合わせた自然なコメントにする
- コメント投稿前に内容を確認（誤送信防止）

**5. 活動記録**
- いいね・コメントの件数と対象をlearning.dbに記録
- 効果的だったコメントパターンを蓄積

### 注意事項
- 深夜（23:00-8:00）のエンゲージメント活動は避ける
- 同じユーザーに毎日コメントしない（ストーカー的に見える）
- 宣伝的なコメント（「私のROOMも見てね」等）は絶対にしない
- アカウントBANリスクを避けるため、1日の総活動数は15件以内

## トラブルシューティング

### pip / パッケージが無い
```bash
pip3 install --break-system-packages -r requirements.txt
```
注意: コンテナのPythonはPEP 668準拠のため `--break-system-packages` フラグが必要。

### ブラウザ操作が失敗する
- `collect_trends.py`, `collect_competitor.py` はブラウザ依存。失敗したらスキップ可
- ランキングはAPI経由（`collect_ranking.py`）で確実に取得できる

### 楽天APIがエラーを返す
- レート制限（1秒間隔）を守っているか確認
- `.env` の `RAKUTEN_APP_ID`, `RAKUTEN_APP_SECRET` が正しいか確認

### 画像生成が失敗する
- Gemini APIキーが有効か確認（`.env` の `GOOGLE_API_KEY`）
- フォールバック: 商品ページからダウンロードした画像をそのまま使用

### learning.dbが破損した場合
- `data/learning.db` を削除すると自動再作成される（蓄積データは失われる）
